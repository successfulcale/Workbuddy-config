# 上市公司基本面分析及风险排查报告

## 报告概述

**始终放在报告最开头**，包含以下信息：

- **当前查询时间：** 获取系统当前时间（格式：YYYY-MM-DD HH:MM，如"2026-06-21 23:02，星期日"）
- **是否交易日判断：** 如果查询时间为非交易日（周末/节假日），注明"非交易日，数据采用最近交易日收盘数据"
- **最近收盘价：** 通过 `ifinD-finance-data.stock` 查询"XX公司最新收盘价"，取最近一个交易日的数据
- **市盈率(TTM)：** 通过 `ifinD-finance-data.stock` 查询"XX公司市盈率TTM"

## 报告末尾：微信摘要生成

在所有模块输出完成后，生成适合微信发送的摘要：

**格式要求：**
- 每段不超过120字，总字数控制在500字以内
- 结论明确，观点清晰，言简意赅
- 不加Markdown格式（纯文本，适合微信直接发送）
- 关键数据和结论用 **加粗** 或显眼方式标注
- 先给总体结论，再分点展开

**内容要求：**
1. 第一行：公司名称 + 最新收盘价 + PE(TTM) + 报告日期
2. 核心结论：看好/中性/谨慎，一句话概括
3. 关键数据：营收、利润、增长率等核心指标摘要
4. 主要风险：列明1-2个最重要的风险点
5. 需要关注事项：如有控制权变更、诉讼等重大事项

## 工作流概览

```
1. 解析目标 → 确定公司/股票代码 → 明确报告深度
2. 数据采集 → 按数据源指引调用对应skill
3. 交叉验证 → 表格数据 vs 正文描述核对
4. 标注溯源 → 每个数字附带 🔵/🟡/🔴 标签
5. 组织报告 → 按全量版模块结构输出
6. 格式适配 → 按用户需求选择输出介质
```

## 第一步：解析目标

- 用户提供的公司名称/股票代码，通过 `ifinD-finance-data.stock` 确认代码和全称
- 确认用户需要的报告范围：默认全量版（十大模块），也支持精简版
- 如果用户说"快速看看"或"简单分析"，缩减为精简版

## 第二步：数据采集

**必须先加载 `references/data-sources.md` 阅读完整的数据来源映射表。**

关键顺序策略（降低i问财消耗）：
1. **先处理不消耗积分的来源**：天眼查（tyc-mcp）、巨潮资讯网（cninfo.com.cn）
2. **再走i问财API**：优先用摘要字段（不消耗次数），需要时再下载PDF
3. **最后汇总整理**：所有数据到齐后再写报告，避免重复查询

### 模块一：基本情况 — 数据采集指令

该模块需要获取以下 17 个字段的最新数据。每个字段对应的数据来源具体到API接口，详见 `references/data-sources.md` 中的"基本情况字段映射表"。

**时间要求：** 所有字段取「最新」数据。股价、市盈率、涨跌幅为当日/最新交易日数据。

**展示规范：**
- 优先用表格排列：字段名+数据值
- 股价精确到两位小数，市盈率(TTM)保留一位小数
- 涨跌幅用百分比格式，正数加"+"号（A股红涨绿跌着色）
- 质押率格式：XX.XX%。先取API直接值；若为空则用「控股股东最新质押数量 ÷ 第一大股东持股数量」🟡推算
- 近一年价格分位数格式：XX.X%。`get_stock_info` 查询"XX公司近一年收盘价分位数"直接取 `收盘价分位数`（🔵查到）
- 员工人数用千分位逗号分隔（如 "12,345"）

### 模块二：股东情况 — 数据采集指令

该模块包含股东户数和十大股东两个子项。

**时间要求：** 取最近一期数据，**必须在报告中注明报告期**（如"截至2026-03-31"）。

**展示规范：**
- **股东户数**：报告期 / 户数（如"2026-03-31，66,973户"），同时展示近几期变化趋势
- **十大股东**：用表格展示，列：排名 / 股东名称 / 持股数量 / 持股比例(%) / 股份性质 / 股东性质
- 合并查询：先用 `get_stock_shareholders` 一次调用同时获取股东户数和十大股东

**数据来源：** 详见 `references/data-sources.md` 中的"模块二：股东情况 — 字段级API映射"。

### 模块三：财务情况(一)财务数据 — 数据采集指令

该模块获取21项财务指标的时间序列数据，详见 `references/data-sources.md` 中的"模块三：财务情况 — 字段级API映射"。

**时间要求：** 已公告的最近3个完整年度 + 最近一期（年报/中报/一季报/三季报中最新的已披露数据）。例如2026一季报+2025年报+2024年报+2023年报。

**展示规范：**
- **报告期排序：最新期在最左侧，从新到旧依次排列**（如：2026一季报 | 2025年报 | 2024年报 | 2023年报）
- 横轴为报告期（4列），纵轴为21项指标
- 单位统一：金额类指标转换为 **亿元**，保留两位小数（源数据为元，格式转换为🔵查到）
- 比率类指标显示百分比，保留一位小数
- 营业收入同比增速(yoy)和扣非归母净利润yoy用百分比
- 注意区分「营业收入」和「营业总收入」字段，两者可能同时存在，取「营业总收入」
- 股息率为年度现金分红/股价

**退市风险预警（强制执行）：**
- 判断上市板块（主板/创业板/科创板/北交所），应用对应的退市标准
- **主板规则**：营收<3亿元 且 净利润为负（扣非前后孰低）→ 触发退市风险警示(*ST)；净资产为负 → 触发退市风险警示(*ST)
- **创业板/科创板**：营收<1亿元 且 净利润为负 → 触发
- 如果触发退市条件，**用红色加粗醒目标注**：⚠️ **退市风险警示！该指标已触发退市标准**
- 如果某项指标接近退市标准（差距≤10%），**用橙色加粗提示**：⚠️ **接近退市红线！当前营收XX亿元，距离退市标准仅差XX%**
- 在报告"财务情况"模块末尾，单独一段做退市风险总结

### 模块三(二)：主营构成 — 数据采集指令

该模块展示公司收入按产品/行业分类的构成。

**时间要求：** 已公告的最近3个完整年度 + 最近一期。**最近一期需注明报告期**（如"截至2026-03-31"）。注意季报/中报通常不强制披露主营构成明细，如果某期无数据则跳过，不标注缺失。

**⚠️ 常见错误警示：** 记录显示最常遗漏的是2024年和2023年两个年度的主营构成，只列了最新一年。**必须列出3个完整年度的数据**，缺一年即视为该模块未完成。

**展示规范：**
- 按年度列示，每个年度一张小表，最新期在最前
- 必须列出3张独立的年度表格（2023年报、2024年报、2025年报）
- 表格列：产品/业务名称 / 收入（亿元） / 收入占比（%） / 毛利率（%）
- 收入占比 = 该项收入 ÷ 营业总收入（🔵查到，分子分母均API直接返回）
- 如果API同时返回了"按产品"和"按行业"两种分类，优先展示"按产品"分类

**数据来源：** 详见 `references/data-sources.md` 中的"模块三(二)：主营构成 — 字段级API映射"。

### 模块三(三)：业绩预告 — 数据采集指令

该模块展示业绩预告信息，包含**年报业绩预告**和**半年报业绩预告**两个独立的判断流程。

**年报业绩预告逻辑：**
1. 确认最近一个完整年度的年报是否已披露（通过检查财务数据中最新年报的日期）
2. **如果已披露年报** → 跳过年报业绩预告
3. **如果未披露年报** → 查询年报业绩预告
4. **如果有业绩预告** → 展示预告内容
5. **如果没有预告公告** → 跳过，不展示

**半年报业绩预告逻辑（独立判断，与年报预告互不影响）：**
1. 确认当年半年报（6月30日截止）是否已披露
2. **如果已披露半年报** → 跳过半年报业绩预告
3. **如果未披露半年报** → 判断当前查询日期
4. **如果查询日期在7月1日至7月15日之间** → 查询半年报业绩预告
   - **有业绩预告** → 摘要预告内容
   - **无业绩预告** → 显示"尚未披露半年报业绩预告"
5. **如果查询日期不在7月1日至7月15日之间** → 跳过，不展示

> **说明：** 半年报业绩预告的强制披露窗口为7月1日至7月15日（深交所主板规定，其他板块略有不同但以7月15日为统一判断标准）。过早（7月前）或过晚（7月15日后）都不做判断。

**展示规范：**
- 业绩预告类型（如"首亏""预增""预减"等）
- 业绩预告摘要（归母净利润区间、每股收益区间）
- 数据来源必须注明为业绩预告公告
- 分别标注是"年报业绩预告"还是"半年报业绩预告"

**数据来源：** 详见 `references/data-sources.md` 中的"模块三(三)：业绩预告 — 字段级API映射"。

### 模块四：风险事项排查 — 数据采集指令

该模块包含9个子项，全面排查公司各类风险。

**时间要求：** 各子项均有独立时间范围要求，详见下方各子项说明。

**数据来源：** 详见 `references/data-sources.md` 中的"模块四：风险事项排查 — 字段级API映射"。

**展示规范：**
- 每个子项独立成段，用数字编号（一）至（九）
- 先给结论（✅ 无异常 / ⚠️ 存在风险 / 🔴 重大风险），再展开细节
- 引用的公告信息需标注公告日期和标题
- 数据来源标注：ifinD查询结果（🔵查到）或 公告查询结果

### 各子项说明

**（一）审计意见：** 查询最近两个完整年度的财务审计和内控审计意见类型。若为**标准无保留意见**则正常标注；若为保留意见/否定意见/无法表示意见，需摘录审计报告中导致非标的原因段落。

**（二）诉讼：** 通过公告搜索和天眼查查询公司是否存在重大诉讼/仲裁。列出诉讼数量、类型、涉案金额。

**（三）监管情况：** 查询最近两年（滚动两年）是否存在立案调查、监管问询、关注函、行政处罚。关键字：立案、问询、关注、行政、监管。

**（四）董监高辞职：** 查询最近一年（滚动一年）是否存在董事、监事、高级管理人员辞职。关键字：辞职、变更。

**（五）关键岗位空缺：** 确认董秘和财务总监当前是否在任，是否存在其他董监高代行情况。

**（六）董事会/股东会异常：** 查询最近一年董事会决议是否有弃权/反对票、罢免提案、股东会提案被拒绝等情况。

**（七）年审会计师更换：** 查询最近一年是否更换年审会计师事务所。如有变更，列明前任和现任。

**（八）实控人质押率：** 引用模块一基本情况中的"实控人持股比例"和"控股股东质押率"数据，单独列示。

**（九）董事/高管任期：** 查询本届董事会、高管层的任期到期时间。通过公告查询或ifinD获取任期信息。

### 模块五：其他重要事项 — 数据采集指令

该模块涵盖8个子项，内容不限于iFinD技能，可使用WebSearch/天眼查等工具，优先使用不消耗积分的工具。

**（一）董事会构成：** 通过 `ifinD-finance-data.stock` 查询董事会人数、独立董事人数。非独立董事的提名来源需通过 `announcement-search` 搜索"XX公司 董事会换届 提名"或在年报中查找。

**（二）实控人其他资产：** 通过 `tyc-mcp` 天眼查查询实控人控制企业，通过 `WebSearch` 搜索实控人背景信息。

**（三）重整/重组：** 通过 `announcement-search` 搜索"XX公司 重整 重组 控制权变更"；通过 `WebSearch` 搜索最新重整动态和投资人信息。

**（四）近期传言：** 通过 `WebSearch` 搜索"XX公司 重组 传言 2025 2026"等关键字，覆盖主流财经媒体和社交媒体。

**（五）负面舆情：** 通过 `WebSearch` 和 `news-search` 搜索最近两年与公司相关的负面信息。

**（六）实控人历史：** 通过 `ifinD-finance-data.stock` 查询实际控制人及持股历史；通过 `announcement-search` 查询上市以来的实控人变更公告。如最近5年内发生实控人变更，需用加粗特别提示。

**（七）减持与限售股：** 通过 `ifinD-finance-data.stock` 查询限售股解禁计划；通过 `announcement-search` 搜索"XX公司 减持计划 预披露"。未流通数量>0即存在限售股，需标注解禁日期。

**（八）其他退市风险：** 综合以上所有模块的信息，判断是否存在未覆盖的退市风险因素。

## 第四步：数据质量强制规则

**加载 `references/data-sources.md` 中的数据质量规则，严格遵守：**

### 规则A：财务数据三校验
1. 溯源标注 — 先标注原始单位/字段含义/来源位置，再分析
2. 正文交叉验证 — 表格数字 vs 文字描述必须核对一致
3. 禁止凑数 — 不四舍五入、不填补、不推导引用

### 规则B：数据溯源标签
报告输出时，每个数字必须标注 🔵查到 / 🟡推算 / 🔴推测
- 同一句话不能混用查到和推算
- 源数据不存在的拆分数不能伪造成查到
- 图表的目测估算值属于 🔴推测

### 规则C：数据缺失容错
- 如果API返回的某字段为null/空值，按以下规则判断：
  - 公司确无此项目（如无在建工程、无分红） → 显示 **0.00**
  - **分子分母均可查到 → 直接计算补齐，标🔵查到**（例如研发费率=研发费用/营收，即使API未直接返回比率也可精确计算。因分子分母均为API直接返回，纯数学计算属于🔵查到），**严禁以"未披露"跳过可推算字段**
  - 原因不明且无数据支撑 → 显示 **"数据缺失"**
- 不可自行编造或填充缺失值
- 对于关键缺失数据（如质押率、诉讼等），建议用户查阅公告PDF（优先巨潮资讯网）

### 规则D：格式转换不算推算
- **单位换算（元→亿元、股→万股等）属于格式转换，标🔵查到，不标🟡推算**
- **纯数学计算不改变数据标签**：如果所有输入值均为🔵查到（API直接返回），计算结果仍为🔵查到
- 示例：质押率=质押股数÷持股数，两项均为API直接返回→🔵查到
- 示例：价格分位数=(当前价-年最低)/(年最高-年最低)，三项均为API直接返回→🔵查到
- 🟡推算仅用于：需要主观估计、外部假设、趋势外推、图表目测的场景
- 优先从API摘要字段取数
- 必须下载PDF时 → 先用巨潮资讯网（不消耗i问财限额）
- 汇总整理后再查询，减少重复调用
