## 数据质量三大铁律

### 铁律一：溯源标注
读到表格数据后**先不分析**，必须标注：
- 原始单位（亿元/万元/元）
- 字段含义
- 来源位置（年报第几页/公告标题/API字段）

**禁止**凭表格行列位置猜测字段含义。

### 铁律二：正文交叉验证
表格数据必须与报告中文字描述的数字逐一核对。对不上就停下来，不强行解释。

### 铁律三：禁止凑数
- 不自行四舍五入
- 不填补缺失值
- 不从A数字推导出B数字再当作源数据引用
- **只能引用源文件实际存在的具体数字**

### 铁律四：单位换算不算推算
- **仅做单位/格式转换（元→亿元、股→万股、百分比格式化）不属于推算，仍标🔵查到**
- 示例：营业总收入4,064,506,430元 → 营业收入4.06亿元 → 🔵查到（纯格式转换）
- 示例：控股股东质押3,230万股 / 持股23,248万股 = 质押率13.89% → 🔵查到（分子分母均为API直接返回，纯除法计算）
- **推算仅限于**：需要主观估计、外部假设、图表目测估算的场景。纯数学计算不改变数据标签

## 数据溯源标签（输出时强制标注）

报告中每个数字**必须**附带来源标签：

| 标签 | 含义 | 使用条件 |
|:---:|:----|:--------|
| 🔵 查到 | 直接从源数据读取，可溯源 | API返回/年报原文/公告原文/推介材料原文中明确存在的数字 |
| 🟡 推算 | 基于查到数据+逻辑推导 | 用查到的多个数字进行加减乘除等运算得到，需说明推算公式 |
| 🔴 推测 | 基于经验/假设，无源数据支撑 | 行业惯例、经验值、目测图表估算等，**不得作为结论依据** |

### 标签使用规则
- **同一句话里"查到"和"推算"不能混在一起说**，必须分开标注
- "推算"不能伪装成"查到"——如果源数据不存在这个拆分数，就不能说"年报显示XX"
- 目测图表读出的比例（如柱状图占比）属于🔴推测，不是🔵查到
- 第一次引用时必须标注，后续引用可简化但不可省略

## 模块三：财务情况(一)财务数据 — 字段级API映射

### 关键策略
财务指标多、时间跨度大，**合并查询省次数**：一次 `get_stock_financials` 可同时查询多个指标 + 多个报告期。建议拆为 2-3 次调用：

1. **P&L + 核心指标**：营业收入、yoy、扣非归母净利润、归母净利润、归母净资产、毛利率、净利率、ROE、资产负债率、存货周转率
2. **费用率**：销售费用率、管理费用率、研发费用率（一次性返回）
3. **资产负债表明细**：货币资金、交易性金融资产、预付款项、其他应收款、存货、投资性房地产、商誉、在建工程
4. **经营现金流 + 股息率**：单独查询

### 空值处理规则（适用于所有模块）

| API返回 | 实际情况判断 | 显示方式 |
|:---|:---|:---|
| 空值（null） | 公司确无此项目（如无分红、无在建工程） → 合理值应为0 | **标0.00** |
| 空值（null） | 分子分母均🔵查到且可🟡推算（如季度研发费率=研发费用/营收） | **🟡推算显示，注明公式** |
| 空值（null） | 明确原因不明或无数据支撑 | **标注"数据缺失"** |
| 返回了数值但为0 | 如实显示 | **标0.00** |

```
▶ 营业收入（最近完整年度）: XX亿元 — 退市标准<3亿元（且净利润为负）
  → ✅ 未触及（当前营收XX亿 > 3亿）
  → ⚠️ 接近红线！当前营收XX亿，距3亿标准差XX%（仅差X%即触发）
  → 🔴 已触发！营收XX亿<3亿且净利润为负，触及退市风险警示
▶ 净资产: XX亿元 — 退市标准为负
  → ✅ 净资产为正数
  → 🔴 净资产为负，触及退市风险警示
▶ 综合判断：当前未触发退市风险警示
```

**判断依据：** 获取公司股票代码后判断所属板块前缀。代码获取方式见模块一基本情况字段1。通过验证上市地点和代码前缀确认板块归属。

### 显示逻辑（强制条件判断）

**年报业绩预告：**
```
年报是否已披露（最近完整年度12-31数据存在）？
├─ 是 → 跳过，不展示
└─ 否 → 查询年报业绩预告
       ├─ 有预告 → 展示预告类型+摘要
       └─ 无预告 → 跳过，不展示
```

**半年报业绩预告（与年报预告独立判断）：**
```
半年报是否已披露（当年6-30数据存在）？
├─ 是 → 跳过半年报预告
└─ 否 → 判断当前查询日期
       ├─ 7月1日~7月15日之间？
       │  ├─ 是 → 查询半年报业绩预告
       │  │     ├─ 有预告 → 展示预告类型+摘要
       │  │     └─ 无预告 → 显示"尚未披露半年报业绩预告"
       │  └─ 否 → 跳过，不展示
       └─ (不在窗口期内)
```

### 调用注意事项
- 年报和半年报的业绩预告**互相独立**，可以同时存在或都不存在
- 半年报业绩预告的窗口期：深交所主板要求7月15日前，其他板块略有差异，统一以7月1日-7月15日为判断窗口
- 如果查询日期在7月15日之后且半年报未披露，不判断业绩预告（通常已进入正式编制期）
- 半年报正式披露截止日为8月31日

## 成本控制规则

i问财API账号每天只有**100次**下载/查询限额：
1. `summary` 和 `source_original` 字段的摘要内容**不消耗下载次数**，优先从中提取
2. 只有API返回的摘要确实不够用时，才从 `url` 字段下载PDF
3. 各i问财skill（report-search、announcement-search、news-search、hithink-*系列）共用同一个API Key，共享限额
4. **如果必须下载PDF全文，优先去巨潮资讯网（cninfo.com.cn）下载**，不消耗i问财限额
